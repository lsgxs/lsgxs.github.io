Hugo站点几个主要的设置项目

hugo站点几个主要的设置项目 在设置好站点配置之后，经常打交道的主要就是下面几个项目 目录结构 这里经常使用的也就是content、staict\images、config.toml。如果不经常更换theme，也不需要修改config.toml的内容。 新建markdown文档 在windows的cmd窗口，使用hugo new posts/xxxx.md来新建markdown文档。生成的.md文件保存在./content/posts目录下。 保存图像文件 需要添加在文档边栏或者在文章内部引用的图片，可以保存在./static/images目录下。新建的hugo站点项目，static目录是空的，需要自己在static目录下新建images目录 上传到github 在config.toml文件中使用publishdir=“path"来指定hugo -D的发布路径。我这里没有使用默认的public，config.toml设置指向的是togithub目录，已经使用git init初始化了，发布完成后，直接使用 add . git commit -m "xxx" git push origin main 推送到github。 试了不少的theme，最终上传到github仓库时，打开username.github.io时在浏览器里渲染错位。也不想再试了，从hexo到hugo,再到别的静态生成器，学的都是套路，想要更深入的话，还是html+css+javascript。 关于使用F12在浏览器里调试最终渲染的源代码。 浏览器打开博客网页，按下f12，然后选择工具栏左侧第一个“在页面中选择一个元素来检查”按钮，然后再页面中点击需要查看源码的元素，源代码区域就会显示最终的运行时源码，查找错误很方便。
